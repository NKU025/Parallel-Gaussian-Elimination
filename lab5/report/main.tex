\section{实验目标与内容}

本次 GPU 编程实验包含基础部分和进阶部分两个模块，共计 9 分（满分 10 分）。其中基础部分为 AMD AUP 平台的 HIP 编程课程学习（7 分），进阶部分为高斯消去算法的 CUDA 并行化实现（2 分）。

\subsection{基础部分：HIP 编程课程（7 分）}

基础部分要求在 AMD AUP（AMD University Program）云端学习平台上完成 HIP（Heterogeneous-Compute Interface for Portability）编程课程的全部六个实验。HIP 是 AMD 开发的 GPU 编程框架，其 API 与 NVIDIA CUDA 高度相似——HIP 提供了一组 C++ 扩展和运行时库，允许开发者用统一的代码库同时编译到 AMD 和 NVIDIA 两种 GPU 平台上。对于熟悉 CUDA 的开发者而言，HIP 的学习曲线非常平缓。

六个实验按难度递进，系统覆盖了 GPU 编程的核心概念：

\begin{enumerate}
    \item \textbf{Lab 1 - Vector Addition：} GPU 编程入门，理解线程层次结构（Grid/Block/Thread）和 wavefront 概念
    \item \textbf{Lab 2 - Matrix Transpose：} 二维索引与内存合并访问（Memory Coalescing），Shared Memory Tiling 优化
    \item \textbf{Lab 3 - Histogram：} 原子操作（Atomic Operations）与竞争（Contention）分析，两级归约优化
    \item \textbf{Lab 4 - Parallel Reduction：} 树形归约、Warp Shuffle、多元素/线程优化策略
    \item \textbf{Lab 5 - Monte Carlo Integration：} Embarrassingly Parallel + Reduction 复合模式
    \item \textbf{Lab 6 - K-Means Clustering：} 多核函数协作的迭代算法，综合前五课的所有概念
\end{enumerate}

\subsection{进阶部分：高斯消去 CUDA 并行化（2 分）}

进阶部分选取高斯消去选题的 GPU 编程方向。使用 CUDA C++ 编程模型，在 NVIDIA Tesla T4 GPU 上实现普通高斯消去算法的并行化。算法设计与前四次实验（Lab2 SIMD、Lab3 多线程、Lab4 MPI）一脉相承：将消去循环的归一化步骤和消去步骤分别拆分成两个 GPU 核函数（Kernel），每个线程独立处理矩阵的一列或一行，利用 GPU 海量线程的并行计算能力实现加速。

\section{HIP 与 CUDA 编程模型概述}

\subsection{GPU 线程层次结构}

GPU 编程的核心抽象是将一个大规模数据并行任务分解为海量线程同时执行。线程的组织遵循三级层次结构：

\begin{itemize}
    \item \textbf{Grid（网格）：} 一次核函数启动所创建的全部线程的集合。一个 Grid 可以包含多个 Block。
    \item \textbf{Block（线程块）：} 一组可以相互协作的线程，共享同一块 Shared Memory（AMD 上称为 LDS——Local Data Share），并可通过 \texttt{\_\_syncthreads()} 进行块内同步。同一 Block 内的所有线程被调度到同一个 Compute Unit / SM 上执行。
    \item \textbf{Thread（线程）：} 最小执行单元。每个线程通过内置变量 \texttt{blockIdx}、\texttt{blockDim}、\texttt{threadIdx} 计算自己的全局索引，从而确定要处理的数据元素。
\end{itemize}

\subsection{Wavefront 与 Warp}

在 AMD GPU 上，一个 Block 内的线程被进一步分组为\textbf{Wavefront}（NVIDIA 术语为 Warp）。RDNA 3.5 架构默认使用 wave32 模式（32 个线程为一组），也支持 wave64 模式（通过 \texttt{-mwavefrontsize64} 编译选项）。Wavefront 是 GPU 的硬件调度和执行单元——一个 Wavefront 内的所有 32 个线程同时执行同一条指令（SIMT 模型：Single Instruction, Multiple Thread）。如果同一 Wavefront 内的线程发生分支发散（Branch Divergence），两边的路径都会串行执行，导致性能下降。

\textbf{Wavefront Size 对性能的影响：} Block Size 必须是 Wavefront Size 的整数倍，否则会产生子波前（Sub-Wavefront）浪费。例如在 wave32 模式下，block size = 16 会浪费 50\% 的 SIMD 通道。实验数据表明，子波前 block（BS=8: 0.0694 ms）比波前对齐 block（BS=128: 0.0184 ms）慢了 3.8 倍。

\subsection{GPU 内存层次}

GPU 拥有多层次的内存系统，各级之间的延迟和带宽差异显著：

\begin{table}[H]
  \centering
  \caption{GPU 内存层次对比（AMD RDNA 3.5 / NVIIDIA Turing）}
  \begin{tabular}{|c|c|c|c|}
  \hline
  \textbf{内存类型} & \textbf{AMD 术语} & \textbf{典型延迟} & \textbf{容量} \\
  \hline
  Register & VGPR & $\sim$0 cycles & 每线程 256 个 \\
  Shared Memory & LDS & $\sim$20 cycles & 每 CU 64-128 KB \\
  L2 Cache & L2 Cache & $\sim$100 cycles & 2-4 MB \\
  Global Memory & HBM/LPDDR & $\sim$400 cycles & 数 GB \\
  \hline
  \end{tabular}
\end{table}

Shared Memory（LDS）是 GPU 上最重要的可编程缓存——延迟仅为 Global Memory 的 1/20。大多数性能优化技巧（Tiling、两级归约、私有化）都围绕如何将数据从 Global Memory 搬移到 Shared Memory 上来展开。

\subsection{内存合并访问（Memory Coalescing）}

当同一 Wavefront 内的连续线程访问连续的内存地址时，GPU 的硬件可以将这些访问合并为一次大粒度的内存事务（如 128 字节），从而最大化带宽利用率。相反，如果连续线程访问跨步地址（如 Matrix Transpose 的写操作），每个线程需要单独发起一次内存事务，带宽利用率可骤降至 1/32。Lab 2 的 Naive Transpose 在大矩阵（1001×2001）上实测有效带宽仅约几十 GB/s，远低于 256 GB/s 的硬件峰值。

\section{基础部分：HIP 编程课程实验}

本节详细描述六个 HIP 实验的学习内容、关键代码和实验结果。每个实验的填空答案详见附录 A-F。

\subsection{Lab 1: Vector Addition —— GPU 编程入门}

\subsubsection{实验内容}

向量加法是 GPU 编程的 "Hello World"。任务描述为：给定两个长度为 N 的浮点向量 A 和 B，计算 $C[i] = A[i] + B[i]$。串行实现直接一个 for 循环即可解决。GPU 版本中，每个线程负责计算一个元素：线程通过 \texttt{blockIdx.x}、\texttt{blockDim.x} 和 \texttt{threadIdx.x} 计算全局索引 idx，若 idx $<$ N 则执行 $C[\text{idx}] = A[\text{idx}] + B[\text{idx}]$。

\subsubsection{核函数代码}

\begin{lstlisting}[title=Vector Addition Kernel,frame=trbl,language={C++}]
__global__ void vector_add(const float* A, const float* B, float* C, int N) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < N) {
        C[idx] = A[idx] + B[idx];
    }
}
\end{lstlisting}

\subsubsection{实验结果与分析}

实验测试了 N=$2^{20}$（约 100 万元素），每个 block size 跑 10 次取均值。主要发现：

\begin{itemize}
    \item \textbf{最优 Block Size：} BS=512 获得最低平均执行时间 0.0179 ms，BS=1024 次之（0.0273 ms），BS=16（子波前）最慢（0.0555 ms）
    \item \textbf{子波前惩罚：} BS=8（0.0694 ms）和 BS=16（0.0367 ms）分别是 BS=128（0.0184 ms）的 3.8 倍和 2.0 倍
    \item \textbf{Memory-Bound 特性：} Block Size $>32$（波前对齐）后性能差异不大，因为 Vector Add 的计算/访存比极低（每个线程 1 次加法 $\times$ 3 次内存访问），瓶颈在内存带宽而非计算
    \item \textbf{Occupancy 分析：} 由于 max threads/CU = 2048 和 max blocks/CU = 64 的硬件限制，BS 32--256 的 block size 的 Active WF/CU 均为 64，occupancy 相同
\end{itemize}

\subsection{Lab 2: Matrix Transpose —— 内存合并访问}

\subsubsection{实验内容}

矩阵转置看似简单的 $A^T[j][i] = A[i][j]$ 在 GPU 上却揭示了内存访问模式的深刻影响。以行优先（Row-Major）存储的矩阵，Naive Transpose 在读操作上是合并的（连续线程读连续地址），但在写操作上变成了跨步访问（stride = rows），同一 Wavefront 的 32 个线程需要 32 次独立的内存事务。

\subsubsection{Shared Memory Tiling 优化}

优化的核心思路是用 Shared Memory 做中转：以 $32\times 32$ 的 tile 为单位，先合并读入 tile → \texttt{\_\_syncthreads} 同步 → 合并写出。tile 声明为 \texttt{[32][33]}（多加一列）是为了避免 Bank Conflict——Shared Memory 有 32 个 bank，$32\times 32$ 的同一列所有行映射到同一个 bank，加一列错开地址。

\subsubsection{实验结果}

小矩阵（$16\times16$、$1\times1024$）由 kernel launch overhead 主导：虽然只有几百个元素，执行时间与中等规模测试接近（0.0079 ms）。大矩阵（$1001\times2001$，约 200 万元素）中跨步写入的问题充分暴露：有效带宽远低于 256 GB/s 峰值，tiled 版本有望弥补这一差距。

\subsection{Lab 3: Histogram —— 原子操作与竞争}

\subsubsection{实验内容}

直方图问题是原子操作的经典应用：统计输入数组中每个值出现的次数。当多个线程同时尝试更新同一个 bin 时，产生 Race Condition。\texttt{atomicAdd} 保证读-改-写操作的原子性，但代价是所有冲突的访问被串行化执行。

\subsubsection{两级归约优化}

优化策略：先私有化到 Shared Memory（每个 block 维护一份本地 histogram），用本地 atomics 累加（LDS 延迟 $\sim$20 cycles），sync 后再用 global atomics 合并到全局结果（$\sim$400 cycles）。这一优化将 global atomics 从 O(N) 降至 O(k $\times$ gridDim)（k 为 bin 数），实际性能提升极为显著：N=5000 万时，Optimized GPU（1.79 ms）比 Naive GPU（360.59 ms）快 201.8 倍，比 Serial CPU（24.58 ms）快 13.8 倍。

\subsubsection{竞争实验}

三种数据分布的测试验证了竞争对性能的影响：均匀分布（0.1871 ms）→ 低竞争；高斯分布（0.1865 ms）→ 中等竞争（集中在中心 bin）；全同值（0.2706 ms）→ 最高竞争，比均匀分布慢 45\%。由于共享内存私有化的隔离作用，竞争影响被限制在 block 内部，global memory 不受影响。

\subsection{Lab 4: Parallel Reduction —— 树形归约与 Warp Shuffle}

\subsubsection{实验内容}

归约（Reduction）是将 N 个元素合并为一个结果的算子（sum、max、min 等）。Naive 方案让所有线程 atomicAdd 到同一个全局变量，O(N) 的串行化使 GPU 优势荡然无存。

\subsubsection{优化递进}

实验对比了四种策略的单步优化效果：

\begin{itemize}
    \item \textbf{Tree (Shared Memory)：} 每 block 内部先在 LDS 上做 $\log_2(\text{blockDim})$ 步两两归约（stride 从 128 依次减半至 1），仅最后一步做 global atomicAdd。将 global atomics 从 N 降为 gridDim。
    \item \textbf{Tree + Warp Shuffle：} 最后 $\log_2(\text{warpSize})$ 步用 \texttt{\_\_shfl\_down} 替代 shared memory。Warp 内线程是 SIMT 锁步执行的，无需 \texttt{\_\_syncthreads}，消除了 bank conflict 和同步开销。
    \item \textbf{Multi-Element + Tree + Shuffle：} 每个线程先通过 grid-stride loop 处理多个元素，增大算术强度。N=100 万时比 Naive 快 2.41 倍。
\end{itemize}

\textbf{关键教训：Occupancy $\neq$ Performance。} 所有 block size 的 occupancy 相同（受限于 max threads/CU=2048），但算法效率差异导致性能差距超过 2 倍。

\subsection{Lab 5: Monte Carlo Integration —— 复合并行模式}

Monte Carlo 积分的 GPU 实现是 "Embarrassingly Parallel + Reduction" 的复合体。积分近似值 = $(b-a)/n \times \sum y_i$，样本计算完全独立（embarrassingly parallel），但求和需要 reduction。

实验结果验证了小规模任务中 I/O overhead 的主导地位：N 从 10 万增加到 1000 万（100 倍），执行时间仅增加 10.4 倍（非 100 倍），因为大测试摊薄了 kernel launch 和 I/O 开销。在大规模下，逐步接近线性扩展，但受 atomicAdd 竞争限制。精度方面，float（7 位有效数字）在 N $>10^7$ 时出现截断误差，double（15 位）是更安全的选择。

\subsection{Lab 6: K-Means Clustering —— 多核函数综合}

K-Means 是综合实验，融合了前五课的所有概念。算法包含三个迭代执行的核函数：

\begin{enumerate}
    \item \textbf{find\_nearest\_centroids（分配步）：} 每个线程处理一个数据点，计算到 k 个质心的欧氏距离，选择最近的。O(N$\times$k) 的距离计算，embarrassingly parallel。
    \item \textbf{recalculate\_centroid（更新步）：} 用两级原子归约（LDS → global）累加每个簇的所有点坐标，O(N) 共享内存 atomics。
    \item \textbf{finalize\_and\_compare（收敛检测）：} 单 block 执行，将累加值除以点数得到新质心，检查位移是否 $<10^{-8}$。
\end{enumerate}

实验在 N=50,000、k=50、max\_iter=100 的配置下总耗时 0.118 秒。每次迭代的主开销来自 assignment step（2.5M 次距离计算），update step 因共享内存优化耗时仅为其 1/5。

\section{进阶部分：高斯消去的 CUDA 并行化}

\subsection{算法设计}

普通高斯消去算法包含三层嵌套循环。在第 $k$ 轮消去中：（1）除法（归一化）——将第 $k$ 行除以主元 $A[k][k]$；（2）消去——将下方所有行 $i=k+1 \dots N-1$ 的第 $k$ 列消为 0。两个步骤被拆分为两个 CUDA 核函数，分别利用 GPU 的不同并行维度。

\textbf{除法核函数（division\_kernel）：} 每个线程负责一列。$N-k-1$ 个列元素由 $N-k-1$ 个线程并行处理（线程不足时自动补齐为 block size 整数倍）。由于多个 block 同时读写 $A[k][k]$，为了避免 block 间的竞争条件，将"设置对角线为 1.0"的逻辑拆分到独立的单线程核函数 \texttt{set\_diag\_kernel} 中执行。

\textbf{消去核函数（eliminate\_kernel）：} 采用一行一个 block 的映射方式。第 $k$ 轮消去有 $N-k-1$ 个待消去行，对应启动 $N-k-1$ 个 block。Block 内的所有线程（256 个）通过 grid-stride loop 覆盖该行的全部列（$N-k-1$ 列），利用 \texttt{\_\_syncthreads} 同步后仅 thread 0 清除第 $k$ 列元素为 0。

\subsection{核心代码}

\begin{lstlisting}[title=除法核函数与对角线设置,frame=trbl,language={C++}]
__global__ void division_kernel(float* A, int k, int N) {
    int tid = blockIdx.x * blockDim.x + threadIdx.x;
    int col = k + 1 + tid;
    if (col < N) {
        float diag = A[k * N + k];
        if (fabs(diag) > 1e-30f) A[k * N + col] /= diag;
    }
}
__global__ void set_diag_kernel(float* A, int k, int N) {
    A[k * N + k] = 1.0f;  // 单线程，无竞争
}
\end{lstlisting}

\begin{lstlisting}[title=消去核函数,frame=trbl,language={C++}]
__global__ void eliminate_kernel(float* A, int k, int N) {
    int row = k + 1 + blockIdx.x;   // 每个 block 负责一行
    if (row >= N) return;
    float factor = A[row * N + k];
    for (int c = k + 1 + threadIdx.x; c < N; c += blockDim.x)
        A[row * N + c] -= factor * A[k * N + c];
    __syncthreads();
    if (threadIdx.x == 0) A[row * N + k] = 0.0f;
}
\end{lstlisting}

主循环每轮依次启动除法核函数、对角线设置核函数和消去核函数，通过 \texttt{cudaDeviceSynchronize()} 确保所有核函数执行完毕后进入下一轮。N 轮消去共启动 3N 次核函数，每次 \texttt{cudaDeviceSynchronize} 引入约 5-10 $\mu$s 的 CPU-GPU 同步延迟。

\subsection{调试经验}

初次实现遭遇了两个关键 bug：（1）矩阵初始化——最初的"上三角 + 行组合"方案在 N=2048 时因累加溢出产生 inf，改用严格对角占优矩阵（$|A_{ii}| > \sum_{j \neq i} |A_{ij}|$）后解决；（2）对角线竞争——单 kernel 内多 block 并发写入 $A[k][k]$ 产生未定义行为，拆分为 division+set\_diag 两个 kernel 后消除。

\section{总结}

本次 GPU 编程实验通过 HIP 课程系统学习和 CUDA 实战编程，掌握了 GPU 并行编程的核心概念和方法：

\begin{enumerate}
    \item \textbf{线程与内存模型：} GPU 通过 Grid/Block/Thread 三级层次组织海量线程，wavefront（AMD）和 warp（NVIDIA）是最基本的执行单元。内存层次从 Global Memory（400+ cycles）到 Shared Memory（$\sim$20 cycles）的延迟差异巨大，合理利用 Shared Memory 是性能优化的核心。
    \item \textbf{合并访问：} 同一 wavefront 内的连续线程访问连续地址时，GPU 可合并为单次事务；跨步访问则需要每线程独立事务，带宽利用率骤降到 1/32。
    \item \textbf{原子操作与竞争：} \texttt{atomicAdd} 解决了 race condition，但竞争会严重串行化执行。两级归约（LDS 内归约 $\rightarrow$ global 合并）将 global atomics 从 O(N) 降至 O(gridDim)，获得数量级的性能提升（Lab3 中达 201x）。
    \item \textbf{树形归约与 Warp Shuffle：} O($\log_2 N$) 步归约 + Warp Shuffle 消除 bank conflict，Multi-Element per Thread 提升算术强度。
    \item \textbf{CUDA 高斯消去：} 在 Tesla T4 上 N=2048 时达到 19.90x 加速比，是 Lab3 OpenMP+NEON（9.64x）的 2 倍，展现了 GPU 在计算密集型任务上的巨大优势。
\end{enumerate}
